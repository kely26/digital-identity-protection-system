# Code of Conduct

## Our Standard

This project is built for defensive security and privacy protection. Contributors are expected to be respectful, technically rigorous, and responsible in both code and communication.

Examples of acceptable behavior include:

- giving actionable, evidence-based feedback
- reporting issues responsibly and privately when needed
- keeping discussions focused on defensive security outcomes
- respecting different levels of experience while maintaining a high technical bar

Examples of unacceptable behavior include:

- abusive, harassing, or discriminatory language
- sharing real secrets, credentials, or private personal data in issues or pull requests
- proposing offensive, destructive, or non-defensive use cases for the project
- intentionally misrepresenting test results or scan behavior

## Enforcement

Project maintainers may remove comments, close discussions, or reject contributions that violate this standard.

## Reporting

For sensitive behavior or disclosure concerns, follow the process in [SECURITY.md](SECURITY.md).
