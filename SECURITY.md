# Security Policy

Digital Identity Protection System is a defensive security tool for privacy and identity-risk assessment.

## Supported Use

- Local host auditing
- Credential hygiene analysis
- Browser security review
- Email and phishing inspection
- Defensive reporting and remediation guidance

## Out of Scope

- Offensive exploitation workflows
- Browser credential vault decryption
- Remote breach lookups in the default product path
- Weaponization of findings

## Reporting Issues

If you discover a security defect in the tool itself:

- do not publish working exploit details immediately
- provide a concise reproduction path
- include affected platform details
- include expected vs actual behavior

Treat findings produced by the tool as user-sensitive data. Reports should be handled as confidential security artifacts.

For the broader product stance on privacy, local-first execution, and defensive-only scope, see [docs/security-philosophy.md](docs/security-philosophy.md).
