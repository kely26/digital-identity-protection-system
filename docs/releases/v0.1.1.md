# DIPS v0.1.1

Released: 2026-03-13

## Highlights

- Added `dips doctor` for runtime diagnostics and support-oriented environment validation.
- Added policy-gated scan exits for CI, managed-service workflows, and beta automation.
- Restored GitHub Actions CI with Ubuntu and Windows coverage across Python `3.11` through `3.13`.
- Added operational documentation for beta support and product-style maintenance workflows.
- Refined the GitHub presentation with a repo-safe animated typing analyst scene.

## Operator Impact

- Existing `scan`, `watch`, `dashboard`, and `demo` flows remain backward compatible.
- `doctor` is additive and intended for beta support and rollout verification.
- Policy-gated scans still write reports before returning a non-zero exit code.

## Recommended Verification

```bash
make lint
make test
make build
dips doctor
make smoke
```

## Upgrade Notes

- No configuration migration is required from `0.1.0`.
- Existing commands remain stable.
- CI is now expected to run remotely once GitHub Actions is available for the repository/account.
